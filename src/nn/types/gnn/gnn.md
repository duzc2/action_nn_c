# gnn

这是一个轻量图神经网络后端，面向本工程“结构化定义 -> profiler 生成 -> train/infer 分离构建”的主链路。

## 输入与拓扑

- 输入：固定节点数图的扁平节点特征
- 拓扑：由 `GnnConfig.neighbor_index` 提供固定槽位邻接关系
- 节点是否参与当前样本：由 `node_mask_feature_index` 控制；若为 `GNN_FEATURE_INDEX_NONE`，则默认所有节点参与

## 前向过程

1. 节点特征编码到隐藏状态
2. 多轮消息传递
3. 通过配置的读出策略导出图级输出向量

当前支持的读出策略：

- `GNN_READOUT_GRAPH_POOL`
  - 对所有活动节点隐藏状态做均值池化
  - 适合图级分类/回归等整体读出场景
- `GNN_READOUT_ANCHOR_SLOTS`
  - 根据主/次锚点特征选择两个锚点节点
  - 再结合主锚点的槽位邻居导出输出
  - 适合“围绕局部关注点读图”的场景

## 训练过程

- 支持独立监督训练
- 支持组合图中的输出梯度回传
- 训练与推理共享同一套结构语义和参数布局

## 设计边界

- 这是通用图网络后端，不内建具体业务语义
- “当前点 / 目标点 / 可通行” 等含义应由上层样本特征编码提供
- 示例工程只负责说明一种可行用法，不改变该网络类型在工程中的通用定位
