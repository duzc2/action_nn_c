# C语言类大模型神经网络 (C-LLM-Like DNN) 项目规格说明书

## 1. 项目概述
本项目旨在实现一个基于 C99 标准的、架构上借鉴大语言模型（LLM）但专为**直接控制**设计的深度神经网络系统。
**与传统 LLM 的核心区别**：
- **LLM**: 输入 Token -> 输出 Token 概率分布 (Softmax) -> 采样选一个。
- **本项目**: 输入 结构化指令序列+状态 -> **输出 所有执行部件的具体控制值** (Parallel Actuator Values)。所有输出神经元同时工作，互不排斥。

系统支持“输入-推理-输出-反馈”的闭环，适用于嵌入式机器人（ESP32 GPIO）、PC 游戏控制（模拟键鼠/手柄）等多种场景。

## 2. 关键约束
- **编程语言**：C99 标准，零外部依赖。
- **环境适应性**：无 GPU，受限内存 (ESP32)。
- **线程模型**：**单线程设计**。
- **静态架构**：训练前确定结构与 IO 映射。
- **部署灵活性**：Tokenizer 与 推理引擎解耦。

## 3. 系统架构：类 Transformer/MoE 结构

### 3.1 模块化输入处理
1.  **指令解析 (Tokenizer)**：独立模块，部署位置灵活。
2.  **推理核心 (Inference Core)**：处理 Token 序列和状态向量。

### 3.2 骨干网络 (Backbone)
- **Transformer Block**: 融合指令与状态。
- **MoE**: 稀疏激活专家网络。

### 3.3 输出层：全并行执行头
- **N 个神经元独立对应 N 个执行通道**。
- **用户定义映射**：配置物理含义与激活函数 (Sigmoid/Tanh)。

## 4. 资源评估与辅助工具链

### 4.1 模型规模预测器 (Profiler)
- 输出参数量、FLOPs、内存占用、IO 映射表。

### 4.2 结构定义生成器
- 自动生成 `network_def.h`。

### 4.3 权重内嵌模式 (Weight Embedding - Optional)
为了追求极致的启动速度和零 IO 开销，系统提供**将权重编译进代码**的可选特性。
- **常规模式**：加载 `weights.bin` 文件（适合调试/PC）。
- **高性能模式**：训练器生成 `weights.c` 源文件，包含所有参数的 `const float` 数组定义。
    - **优点**：无需文件系统，无需加载过程，地址硬编码。
    - **注意**：大模型可能导致编译时间过长或编译器内存溢出，用户需根据规模选择开启。

## 5. 详细功能需求

### 5.1 数据与训练
- **数据加载器**：支持 CSV 加载。
- **物理模拟器**：PC 端闭环验证。
- **序列化器升级**：
    - 支持导出二进制 `.bin`。
    - **新增**：支持导出 C 源码 `.c` (e.g., `const float W_L0[] = {...};`)。

### 5.2 业务逻辑与通信协议
- **Tokenizer 模块**：独立 C 模块。
- **通信协议**：支持 Raw String 和 Token Stream 两种模式。

### 5.3 平台适配
- **PC 平台**：模拟键鼠。
- **ESP32 平台**：GPIO/PWM 控制。

## 6. 目录结构规划
```
src/
  core/           # 核心算子
  model/          # 网络架构
  tokenizer/      # 独立 Tokenizer
  io_map/         # 输入输出映射
  tools/          # 资源评估
  train/          # 训练引擎
  platform/       # 平台适配
  utils/          # 序列化 (含 weights.c 生成逻辑)
```
