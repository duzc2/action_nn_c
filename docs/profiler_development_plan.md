# profiler 开发计划（任务拆解）

## 1. 目标

基于现有需求与设计文档，按可执行阶段完成以下交付：

- 用户程序可传入结构化网络定义，驱动 profiler 生成产物
- 生成代码与源码职责清晰分离：生成代码处理可变性，核心算法在 `src/nn/[网络类型]/`
- 训练/推理、保存/加载、Hash 校验、双模式循环、零警告编译全部落地

## 2. 总体原则

- 严格 C99
- 最严格编译校验 + 警告即错误 + 零警告
- 注释比例超过 50%
- 优先流程/过程/结构治理，不以数据补丁为主方案
- 存在设计取舍风险先请示用户

## 3. 阶段拆解

## 阶段 A：接口与数据结构落地

### A1. 固化公共类型与错误码

- 输入：
  - `profiler_interface_draft.md`
  - `profiler_api_contract_draft.md`
- 任务：
  - 落地 `ProfStatus`、`ProfErrorBuffer`、`ProfModulePath`、`ProfOutputLayout`
  - 统一错误输出机制（终端 + buffer + 错误码）
- 输出：
  - 公共头文件（固定模板接口）
  - 错误码枚举与字符串映射
- 验收：
  - 所有模块引用同一错误码定义
  - 错误描述为英文

### A2. 固化网络定义输入结构

- 输入：
  - `network_topology_requirements.md`（结构组成、连接语义）
- 任务：
  - 落地 `NN_NetworkDef` 及子结构
  - 覆盖隐藏层、激活函数、节点级映射、合并策略、嵌套关系
- 输出：
  - 网络定义结构头文件
- 验收：
  - 可表达 0 层隐藏层与复杂跨层连接

## 阶段 B：profiler 核心流程落地

### B1. 输入接入与参数校验

- 输入：
  - `ProfGenerateRequest`
- 任务：
  - 检查空指针、错误缓冲区、输出路径完整性
  - 模块级 `.c/.h` 路径合法性检查
- 输出：
  - 输入标准化结果
- 验收：
  - 非法参数返回 `PROF_STATUS_INVALID_ARGUMENT`/`PROF_STATUS_PATH_INVALID`

### B2. 拓扑校验引擎

- 输入：
  - 标准化网络定义
- 任务：
  - 标识唯一性、端口存在性、连接合法性
  - DAG 校验、嵌套展开合法性、连接合并策略完整性、展开顺序确定性
- 输出：
  - 校验通过标志或首错失败
- 验收：
  - 首个错误即停，错误可定位到子网/节点/连接

### B3. 语义展开、布局映射、元数据生成

- 输入：
  - 校验通过网络图
- 任务：
  - 生成参数布局映射
  - 生成训练/推理/保存/加载共享元数据
  - 计算网络签名 Hash
  - 构建子网类型到序列化/反序列化接口的注册索引
- 输出：
  - 映射结果 + 元数据 + Hash
- 验收：
  - 元数据可被训练/推理/加载一致消费

### B4. 代码生成与模板复制

- 输入：
  - 模块路径映射 + 映射结果 + Hash
- 任务：
  - 生成六类 `.c`：tokenizer / init / load / train / save / infer
  - 复制固定 `.h` 模板并注入网络描述注释
  - 在加载 `.c` 固化 Hash 常量
  - 生成基于注册索引的子网保存/加载调用调度代码
- 输出：
  - 分模块产物写入用户指定路径
- 验收：
  - `.c/.h` 可分开目录存放
  - 未配置路径时报错，不写固定目录

## 阶段 C：`src/nn` 核心算法模块化

### C1. 按网络类型目录重构

- 输入：
  - 当前 `src` 结构
- 任务：
  - 统一落位 `src/nn/[网络类型]/`
  - 每类网络建立训练/推理两套 `.c/.h` 与 `README.md`
  - 在训练实现提供子网序列化接口，在推理实现提供子网反序列化接口
  - 定义新网络类型接入模板，确保新增类型无需改动旧类型实现
  - 定义注册配置文件或注册宏清单模板，确保新增类型无需改动 profiler 主流程
  - 定义 CMakeLists 网络类型开关模板，确保仅启用类型参与编译
- 输出：
  - 规范化目录结构
- 验收：
  - 训练可依赖推理，推理不依赖训练

### C2. 训练/推理双模式接口实现

- 输入：
  - 接口草案中的 `*_run_auto` / `*_step`
- 任务：
  - 训练实现自动循环和单步触发
  - 推理实现自动循环和单步触发
- 输出：
  - 双模式运行接口
- 验收：
  - 两种模式都可独立运行，不得缺失

## 阶段 D：保存加载一致性与安全边界

### D1. 权重保存格式实现

- 任务：
  - 保存参数数据与网络 Hash
  - 保存布局关联信息（通过元数据）
  - 按拓扑顺序调用已注册子网序列化接口完成全网保存
- 验收：
  - 文件结构稳定，支持后续加载验证

### D2. 权重加载强校验实现

- 任务：
  - 每次加载都校验文件 Hash 与代码常量 Hash
  - 继续训练时校验结构签名与布局摘要
  - 按拓扑顺序调用已注册子网反序列化接口完成全网加载
- 验收：
  - 不一致返回 `PROF_STATUS_HASH_MISMATCH` 或 `PROF_STATUS_LAYOUT_MISMATCH`
  - 拒绝继续执行

## 阶段 E：工程质量与验证

### E1. 编译质量门禁

- 任务：
  - 配置最严格告警
  - 开启警告即错误
- 验收：
  - 零警告通过编译（源码 + 生成代码）

### E2. 测试矩阵落地

- 任务：
  - DAG/环路、路径参数、Hash 混用、双模式循环、继续训练一致性
  - 模块级输出路径分离测试
- 验收：
  - 关键场景均有通过与失败用例

### E3. 文档收敛

- 任务：
  - 更新各 `src/nn/[网络类型]/README.md`
  - 对齐用户手册/开发手册的调用流程
- 验收：
  - 文档与代码一致，调用链完整可复现

## 4. 依赖关系

- A 完成后进入 B
- B3 依赖 B2
- B4 依赖 B3
- C 可与 B 并行推进，但 C2 依赖 A 的接口固定
- D 依赖 B3、B4、C2
- E 贯穿全流程，最终在 D 后做总验收

## 5. 里程碑验收

- M1：完成 A+B，profiler 可接收网络定义并产生产物
- M2：完成 C，`src/nn` 结构与双模式接口落地
- M3：完成 D，保存加载强一致性与防混用闭环
- M4：完成 E，零警告、测试通过、文档收敛

## 6. 风险与应对

- 风险：网络定义字段过多导致实现分散  
  应对：先完成 A2 字段冻结，再推进 B/C
- 风险：生成路径策略与源码目录约束混淆  
  应对：严格区分“源码目录约束”与“生成产物路径参数”
- 风险：双模式循环导致状态机复杂  
  应对：先落地单步语义，再在外层组合自动循环

## 7. 执行要求

- 不引入历史兼容路径
- 不做隐式默认写目录
- 不跳过校验直接继续执行
- 不采用临时数据补丁替代结构化修复
- 正确性优先由流程机制保证，数据验证仅用于校验数据符合流程约束
- 新增网络类型只通过统一配置与注册流程接入，不修改对外接口与旧实现路径
- 新增网络类型接入只允许改注册配置文件或注册宏清单
- 新增网络类型接入需通过 CMakeLists 开关启用，未启用类型不得参与编译
