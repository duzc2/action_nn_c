# profiler 功能模块蓝图（基于当前需求）

## 1. 文档目的

本文基于 `network_topology_requirements.md`，整理需要建立的功能模块，并说明：

- 每个模块做什么
- 模块的工作原理
- 模块需要哪些数据
- 这些数据由谁提供
- 端到端工作流程如何串联

## 2. 模块总览

系统分为两类模块：

- 运行在 profiler 阶段的“分析与生成模块”
- 由 profiler 产出的“业务执行模块（.c）”与固定头文件（.h）

固定原则：

- `.c` 按用户网络描述生成
- `.h` 不按配置生成，从固定模板复制，并附加当前网络描述注释
- 对外 API 头文件保持固定
- profiler 生成代码用于解决网络配置多变性，核心算法固定在 `src` 模块
- 所有网络实现位于 `src/nn/[网络类型]/`
- 每个网络类型目录包含训练与推理两套 `.c/.h`，不能只有单个 `.c/.h`
- 每个网络类型目录包含 `README.md`
- 依赖方向固定为“训练可依赖推理，推理不可依赖训练”
- 源码与生成代码统一遵循 C99 规范

## 3. 分析与生成模块

### 3.1 输入接入模块

- 工作原理：接收用户传入的网络结构体参数，作为唯一输入源，不依赖回调。
- 需要数据：
  - 网络定义对象（子网、连接、节点、端口、类型扩展参数）
  - profiler 运行参数（模块输出路径映射、错误缓冲区指针、缓冲区长度等）
- 数据提供方：
  - 用户程序
- 输出：
  - 标准化后的内部网络表示（供后续校验与生成）

### 3.2 拓扑校验模块

- 工作原理：执行一致性校验并在首个错误快速失败，停止后续流程。
- 需要数据：
  - 标准化内部网络表示
- 数据提供方：
  - 输入接入模块
- 核心检查：
  - 标识唯一性
  - 端口存在性
  - 连接合法性
  - 有向无环（DAG）校验
  - 嵌套展开合法性
- 错误输出机制：
  - 终端输出
  - 写入错误字符串指针
  - 返回错误码
  - 错误描述使用纯英文并可定位来源

### 3.3 语义展开与映射模块

- 工作原理：把网络结构语义展开为可执行映射关系，形成保存/加载/训练/推理共享的内部索引。
- 需要数据：
  - 校验通过的网络表示
- 数据提供方：
  - 拓扑校验模块
- 输出：
  - 参数布局映射
  - 节点连接执行映射
  - 模块级依赖关系

### 3.4 网络签名 Hash 模块

- 工作原理：基于用户网络描述数据结构计算网络签名 Hash，用于防止权重跨网络混用。
- 需要数据：
  - 校验通过且语义稳定的网络描述
- 数据提供方：
  - 语义展开与映射模块
- 输出：
  - 网络签名 Hash 值
  - 可用于写入权重文件与读取时校验的 Hash 信息

### 3.5 代码生成与复制模块

- 工作原理：
  - 依据网络描述生成业务 `.c` 模块
  - 从固定模板复制 `.h` 文件并补充网络描述注释
  - 在权重读取 `.c` 中固化当前网络对应的 Hash 常量
- 需要数据：
  - 映射关系
  - 网络签名 Hash
  - 目标模块清单
  - 各模块 `.c/.h` 输出路径参数
- 数据提供方：
  - 语义展开与映射模块
  - 网络签名 Hash 模块
- 输出：
  - 写入 tokenizer 路径的 tokenizer.c
  - 写入初始化路径的 network_init.c
  - 写入加载路径的 weights_load.c（含 Hash 常量）
  - 写入训练路径的 train.c
  - 写入保存路径的 weights_save.c（含 Hash 常量）
  - 写入推理路径的 infer.c
  - 写入各头文件目标路径的固定模板 .h（含网络描述注释）

## 4. 产出执行模块（.c）的工作原理

### 4.1 tokenizer 模块

- 原理：将输入数据转换为模型需要的内部表示。
- 需要数据：输入格式定义、词表/编码规则（如适用）。
- 数据提供方：用户输入数据与生成代码中的固定规则。

### 4.2 网络初始化模块

- 原理：按网络描述建立运行时结构与参数容器。
- 需要数据：网络拓扑、层参数、默认激活函数与节点覆盖规则。
- 数据提供方：生成代码内嵌描述。

### 4.3 参数保存模块

- 原理：按统一参数布局写出权重，并写入网络签名 Hash。
- 需要数据：参数数组、参数布局、网络签名 Hash。
- 数据提供方：训练状态与生成代码映射。

### 4.4 参数加载模块

- 原理：读取权重文件后先校验 Hash，再装载参数；不一致直接失败。
- 需要数据：文件内 Hash、代码内 Hash 常量、参数布局。
- 数据提供方：权重文件与 weights_load.c 常量定义。

### 4.5 训练模块

- 原理：同时提供“自主循环”与“单步触发”两种执行模式，支持检查点保存、加载后继续训练、版本损失对比。
- 需要数据：训练数据、网络状态、优化配置、检查点路径。
- 数据提供方：训练程序运行参数与生成代码。
- 对外能力：
  - 自主循环入口：模块内部控制完整训练循环
  - 单步触发入口：由外部循环逐次调用单次训练迭代

### 4.6 推理模块

- 原理：同时提供“自主循环”与“单步触发”两种执行模式，加载参数后按统一语义执行推理。
- 需要数据：推理输入、加载后的参数、网络执行图。
- 数据提供方：推理程序与生成代码。
- 对外能力：
  - 自主循环入口：模块内部控制完整推理循环
  - 单步触发入口：由外部循环逐次调用单次推理步骤

## 5. 数据责任分配

### 5.1 用户程序提供

- 网络结构体（完整网络描述）
- profiler 调用参数（各模块 .c/.h 路径、错误缓冲区、长度等）
- 训练与推理的运行输入

### 5.2 profiler 提供

- 校验结果与错误码
- 纯英文可定位错误描述
- 网络签名 Hash
- 分模块 `.c` 代码
- 固定模板复制并注释增强后的 `.h`

### 5.3 权重文件提供

- 参数二进制数据
- 文件内网络签名 Hash

## 6. 端到端工作流程

1. 用户程序构建网络结构体。
2. 用户调用 profiler 显式函数接口并传入结构体。
3. profiler 执行校验；若首错出现，立即报错并停止。
4. 校验通过后，profiler 展开语义映射并计算网络签名 Hash。
5. profiler 生成六类 `.c` 模块，复制固定 `.h` 并加入网络描述注释。
5.1 profiler 按参数写入每个模块各自的 `.c/.h` 目标路径，可分开存储。
6. 用户分别编译训练环境与推理环境（支持分割部署）。
7. 训练程序保存权重时写入 Hash；加载时每次校验 Hash。
8. Hash 不一致则拒绝加载并返回错误码，防止不同网络混用。
9. 训练与推理均可按场景选择自主循环入口或单步触发入口。

## 7. 模块边界约束

- 不保留旧实现入口与兼容分支。
- 不引入回调注册调用模式。
- 不允许动态生成 `.h` 声明结构。
- 不允许绕过 Hash 校验直接加载权重。
- 不允许仅实现自主循环或仅实现单步触发，二者必须同时提供。
- 不允许把核心算法迁移到生成代码，核心算法必须保留在 `src`。
- 不允许破坏训练/推理依赖方向：训练可依赖推理，推理不可依赖训练。
- 不允许缺失网络目录级 `README.md`，每个 `src/nn/[网络类型]/` 必须自说明。
- 不允许使用超出 C99 的语言特性，模板代码与生成代码都必须可在 C99 模式下编译。
